# WordFlow PRD

## 0. 产品概述

### 0.1 产品定位
跨平台轻量生词采集器。专注"存词"一件事，复习交给不背单词等第三方APP。

### 0.2 目标用户
已过考试阶段，日常英文阅读中零散遇到生词，需要高效收集并导入背单词APP的用户。

### 0.3 设计原则
- 极简克制：每个页面只做一件事，不堆功能
- 不背单词风格：低饱和度柔和色调、大量留白、圆角元素、舒适字体层级
- 无广告、无开屏、无社交、无推送轰炸

### 0.4 平台规划
- 第一期：Android APP + Chrome/Edge 浏览器插件
- 第二期：桌面端（Tauri，跨应用采集）+ iOS APP（Action Extension）
- 本文档描述第一期 Android APP + Chrome 插件部分

### 0.5 主题
- 明亮 / 深邃 / 极夜 三种主题
- 支持跟随日出日落自动切换

## 1. 应用结构

### 1.1 页面层级

| 层级 | 内容 | 交互方式 |
|---|---|---|
| 一级 | 词池列表（主界面） | 常驻，APP打开即看到 |
| 二级 | 设置面板 | 点击左上角头像，从底部上滑弹出 |
| 浮层 | 采集确认浮层 | 在其他APP内触发时弹出 |

### 1.2 导航逻辑
- 没有底部Tab导航
- 主界面是唯一的一级页面
- 所有次级内容（设置）从底部上滑弹出
- 采集浮层是独立的系统级浮窗，不属于APP内导航

## 2. 主界面 — 词池

### 2.1 顶部栏

#### 2.1.1 左上角头像
- 显示用户头像（未登录显示默认头像）
- 点击 → 从底部上滑弹出设置面板（见第4节）

#### 2.1.2 中间标题
- 显示APP名称"WordFlow"

#### 2.1.3 右上角功能图标

##### 2.1.3.1 搜索图标
- 点击 → 顶部栏变为搜索输入框
- 实时过滤词池列表，按单词拼写匹配
- 输入框右侧有"取消"按钮，点击恢复原状
- 搜索结果中的匹配字母高亮

##### 2.1.3.2 眼睛图标（显示/隐藏释义）
- 开启状态：所有词同时展开，显示音标和释义
- 关闭状态：只显示单词，无释义（默认状态）
- 点击切换，带轻微动画过渡

### 2.2 词数统计
- 顶部栏下方显示当前词池总词数，格式："▌47 词"
- 实时更新（删除、新增后立刻变化）

### 2.3 词列表

#### 2.3.1 默认展示
- 纯单词列表，每行一个单词，行间有细分隔线
- 白色背景，单词使用深色中等字重字体
- 默认按采集时间倒序排列（最新采集的在最上面）

#### 2.3.2 排序切换
- 支持两种排序：时间倒序（默认）/ 字母升序
- 切换入口待定（可放在词数统计旁或设置面板中）

#### 2.3.3 单词点击展开
- 点击某个单词 → 该行原地展开，显示详细信息
- 展开后该行背景变为浅暖杏色（参考不背单词的选中色）
- 展开内容：
  - 第一行：单词（橙色高亮） + 音标 + 右侧发音按钮 + 删除按钮
  - 第二行：简要中文释义（词性 + 释义）
- 再次点击 → 收起，恢复为纯单词行
- 同一时间只能展开一个词，点击新词时旧词自动收起

#### 2.3.4 发音按钮
- 点击 → 播放单词发音
- 优先使用在线TTS（网络可用时）
- 无网络时不显示或显示为灰色

#### 2.3.5 删除按钮
- 点击 → 弹出确认对话框："确定移除该词？"
- 两个按钮："取消" / "移除"
- 确认移除后：该词从列表中消失，带轻微滑出动画
- 词数统计实时更新

#### 2.3.6 空状态
- 词池为空时，显示居中引导文案
- 文案："在其他APP中选中英文单词，即可存入词池"
- 下方可配简单的操作示意图

### 2.4 多选模式

#### 2.4.1 触发方式
- 长按任一单词 → 进入多选模式

#### 2.4.2 多选模式下的列表
- 每个单词左侧出现圆形勾选框
- 被长按的那个词默认勾选
- 点击其他词 → 勾选/取消勾选
- 顶部栏变为："已选 N 个" + 右侧"全选"/"取消"按钮

#### 2.4.3 多选操作栏
- 底部固定出现操作栏，包含两个按钮：
  - 删除：删除所有选中的词
  - 导出：导出所有选中的词
- 未选中任何词时两个按钮均为灰色不可点击
- 选中一个或以上时按钮高亮可点击

#### 2.4.4 多选删除
- 点击删除 → 弹出确认："确定移除已选的 N 个词？"
- 确认后批量移除，列表和词数统计更新

#### 2.4.5 多选导出
- 点击导出 → 生成文件（格式由导出设置决定，默认txt）
- txt格式：一行一个单词，无其他内容
- 生成后调用系统分享菜单，用户可以保存到文件、发送到微信、或直接用浏览器打开bbdc.cn上传
- 已导出的词不自动删除，但可在列表中变灰或加小标记以区分

#### 2.4.6 退出多选模式
- 点击顶部"取消"按钮 → 退出多选模式
- 按系统返回键 → 退出多选模式

### 2.5 底部导出按钮

#### 2.5.1 常驻显示
- 底部始终显示导出按钮
- 默认灰色（未选词时）

#### 2.5.2 点击行为
- 在非多选模式下点击 → 进入多选模式，提示用户选择要导出的词
- 在多选模式下按钮随选中状态变化（见2.4.3）

## 3. 采集浮层

### 3.1 触发方式

#### 3.1.1 PROCESS_TEXT（系统文本选择菜单）
- 在任何支持系统文本选择菜单的APP中，选中英文单词后，菜单中出现"存入WordFlow"选项
- 点击后弹出采集浮层
- 技术实现：在AndroidManifest中声明PROCESS_TEXT的Intent Filter

#### 3.1.2 无障碍服务取词（可选功能，默认关闭）
- 用户在设置中手动开启
- 开启后可在微信、微博等封闭系统菜单的APP中使用
- 触发方式待定（悬浮球/通知栏/特定手势）

### 3.2 浮层界面

#### 3.2.1 布局
- 系统级浮窗，悬浮在当前APP之上
- 圆角矩形卡片，带轻微阴影
- 背景半透明遮罩（点击遮罩区域等同于点击"取消"）

#### 3.2.2 内容
- 第一行：单词（较大字号，加粗）
- 第二行：音标
- 第三行：简要中文释义（词性 + 释义）
- 底部：两个按钮并排

#### 3.2.3 释义来源
- 本地离线词典（ECDICT，SQLite数据库，约30MB）
- 查询速度要求：100ms以内，确保浮层弹出时释义已就绪
- 如果本地词典查不到：显示"未找到释义"，但仍允许存入

### 3.3 按钮

#### 3.3.1 取消按钮
- 位于左侧
- 点击 → 浮层关闭，不做任何操作，返回原APP

#### 3.3.2 存入词池按钮
- 位于右侧，主色调高亮
- 点击 → 单词存入本地数据库 + 云端同步
- 存入成功后：轻微震动反馈（系统触觉反馈） + 浮层自动收起
- 整个过程（弹出 → 存入 → 收起）不超过2秒

#### 3.3.3 重复词处理
- 如果该词已在词池中 → "存入词池"按钮变为"已在词池中"
- 按钮变灰，不可点击
- 用户只能点"取消"关闭

### 3.4 异常处理

#### 3.4.1 选中内容不是单词
- 如果选中的是一段话或非英文内容
- 尝试提取其中的第一个英文单词
- 如果完全无法识别 → 显示选中的原文，释义区域显示"无法识别为英文单词"
- 仍然提供"存入词池"按钮（用户可能有自己的理由）

#### 3.4.2 选中内容为词组
- 如果选中的是英文词组（如"look forward to"）
- 正常显示，尝试从词典中查询词组释义
- 查不到则显示"未找到释义"

## 5. 数据结构

### 5.1 生词表（本地SQLite）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | INTEGER PRIMARY KEY | 自增主键 |
| word | TEXT NOT NULL | 单词原文 |
| phonetic | TEXT | 音标 |
| definition | TEXT | 中文释义 |
| source_app | TEXT | 来源APP包名（可选记录） |
| created_at | TIMESTAMP | 采集时间 |
| exported | BOOLEAN DEFAULT 0 | 是否已导出过 |
| synced | BOOLEAN DEFAULT 0 | 是否已同步到云端 |

### 5.2 唯一性约束
- word字段唯一，防止重复存入
- 存入前先查询，已存在则在采集浮层显示"已在词池中"

### 5.3 离线词典（ECDICT）

| 字段 | 说明 |
|---|---|
| word | 单词 |
| phonetic | 音标 |
| definition | 中文释义 |
| translation | 中文翻译 |
| pos | 词性 |
| tag | 标签（四级、六级、考研等） |

- 预打包为SQLite数据库，随APP安装
- 约40万词条，数据库文件约30MB
- 只读，不做修改

## 6. 技术要点

### 6.1 Android端

#### 6.1.1 PROCESS_TEXT实现
- AndroidManifest中声明Activity的Intent Filter：
  - action: android.intent.action.PROCESS_TEXT
  - category: android.intent.category.DEFAULT
  - data mimeType: text/plain
- Activity接收选中文本后弹出采集浮层
- Activity主题设为透明/对话框样式

#### 6.1.2 无障碍服务实现（第二优先级）
- 继承AccessibilityService
- 监听文本选择事件
- 获取选中文本后触发采集浮层
- 参考项目：全能分词 (github.com/penglu20/Bigbang)

#### 6.1.3 本地存储
- Room数据库（Android推荐的SQLite封装）
- ECDICT离线词典用独立的SQLite数据库文件

#### 6.1.4 发音
- 优先Android系统TTS引擎
- 备选：在线发音API

### 6.2 云同步（第二优先级）
- 用户注册/登录体系
- 生词数据云端备份
- 多设备同步
- 具体后端方案待定

### 6.3 导出功能
- txt格式：一行一个单词，UTF-8编码
- CSV格式：word, phonetic, definition 三列，UTF-8 with BOM
- 调用系统分享Intent，让用户自行选择保存/发送方式
